# CS330

Final Project Reflection
How do I approach designing software?

I’ve started approaching design by thinking about the end result first, then breaking it down into smaller pieces. For this project, I focused on what I wanted the scene to feel like before worrying about the code. From there, I worked on layout, lighting, and object placement step by step.

One design skill I improved was understanding how visual elements work together. Lighting, spacing, and scale all made a big difference. I followed a simple process of building something basic first, then slowly refining it. That’s something I can use in future projects, especially when working on more complex systems.

How do I approach developing programs?

I approach development by building small pieces, testing them, and then improving them. For this project, I didn’t try to do everything at once. I started with basic shapes, then added textures, then lighting, and kept adjusting from there.

Iteration played a big role. A lot of the final result came from making small changes over time rather than getting it right the first time. My approach has become more structured compared to earlier milestones. I now think more about how things connect instead of just making individual parts work.

How can computer science help me in reaching my goals?

Computer science gives me the ability to build things instead of just thinking about them. With computational graphics, I learned how visuals are actually created and controlled, which is something I can apply in other areas like UI design or interactive tools.

These skills could be useful in my future work, especially if I move toward building applications or working with visual systems. It also helped me get more comfortable with problem-solving and thinking through systems, which applies beyond just graphics.
